# בדיקת זמנים - הפעלה עצמאית

זו אפליקציית React רגילה. הנתונים עצמם (יוזרים, היסטוריית סימונים) נשמרים
מקומית בדפדפן (localStorage) - אין להם קשר לענן. מסד הנתונים ב-Supabase
משמש אך ורק כדי לאפשר **התראות אמיתיות ברקע** (Push) שמגיעות גם כשהאפליקציה
סגורה.

---

## שלב 1: חיבור המפתחות לפרויקט

הפרויקט כבר מגיע עם קובץ `.env.example` שמצביע על פרויקט Supabase קיים
(נוצר במיוחד לאפליקציה הזו, בלי שום קשר ל-Afterglow). כדי להריץ מקומית:

```
cp .env.example .env
```

---

## שלב 2: העלאה ל-GitHub

```
git init
git add .
git commit -m "Check timer app"
git branch -M main
git remote add origin https://github.com/USERNAME/REPO-NAME.git
git push -u origin main
```

**חשוב**: הקובץ `.env` לא יעלה ל-GitHub (מוגדר ב-`.gitignore`) - זה בסדר,
נגדיר אותו בנפרד ב-Vercel בשלב הבא.

---

## שלב 3: פרסום (Vercel) - כ-3 דקות

1. כנס ל-**vercel.com**, לחץ "Sign Up", והתחבר **עם GitHub**
2. "Add New" → "Project"
3. תבחר את ה-repository שיצרת
4. לפני שלוחצים Deploy - תפתח "Environment Variables" ותוסיף את שני הערכים
   מתוך `.env.example`:
   - `VITE_SUPABASE_URL`
   - `VITE_SUPABASE_ANON_KEY`
5. לחץ **Deploy**

---

## איך ההתראות עובדות

בפעם הראשונה שמסמנים "לוקח עכשיו"/"לקחתי לפני", הדפדפן יבקש הרשאת התראות.
אם מאשרים, המכשיר נרשם ל-Push אמיתי - תזכורת עתידית נשמרת בשרת (Supabase),
ופונקציה שרצה שם כל דקה שולחת אותה בזמן הנכון, גם אם האפליקציה סגורה. אם
הדפדפן לא תומך ב-Push או שההרשאה נדחתה, האפליקציה נופלת אוטומטית לתזכורת
מקומית (שפועלת רק כל עוד הטאב פתוח).

**באייפון**: Push עובד רק אם האפליקציה **מותקנת למסך הבית** (שיתוף ← הוסף
למסך הבית), לא כשהיא רק פתוחה בטאב של ספארי.

## מה קורה בעתיד

- כל שינוי עתידי בקוד: `git push` - Vercel מפרסם מחדש אוטומטית תוך דקה
- אין תחזוקה שוטפת - Vercel ו-Supabase חינמיים בהיקף שימוש אישי בלי בעיה
